Blocking I/O

Non-blocking I/O

db.query('select..', function(result){
	//do something with result
});
doOtherOperation();

JavaScript? Why?

"To provide a purely evented, non-blocking infrastructure to script highly concurrent programs"

Design goals

  • Use of callbacks for all I/O
  • Support for the most important network protocols; TCP, DNS, HTTP
  • Familiar for client side JS-programmers and old school Unix-hackers (POSIX)
  • Platform independent

Stream example


var fs = require('fs');
var zlib = require('zlib');

var r = fs.createReadStream('file.txt');
var z = zlib.createGzip();
var w = fs.createWriteStream('file.txt.gz');
r.pipe(z).pipe(w);

HTTP hello world
